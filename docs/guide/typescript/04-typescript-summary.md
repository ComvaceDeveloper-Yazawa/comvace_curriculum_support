# 第4章：TypeScript 入門のまとめ

## ここまでで分かったこと

- TypeScript は「型付き JavaScript」
- 型があるとミスに早く気づける
- `const` と `let` の使い分けが分かる
- 基本の型（string / number / boolean）が書ける
- 配列・オブジェクト・関数にも型を付けられる
- ユニオン型で「どちらか」を表現できる

## ここまでできれば OK のチェック

- 型のエラーが出たら「何が違うか」を読める
- `string` と `number` を混ぜないように書ける
- 関数の引数と戻り値に型を書ける

## 次に進むなら

- `type` と `interface` の使い分け
- 型の再利用（別ファイルに分ける）
- ジェネリクス（より柔軟な型の書き方）

まずは「型を書ける」「型を読める」状態になれば十分です。
次の章で、実務に近い書き方に慣れていきましょう。
