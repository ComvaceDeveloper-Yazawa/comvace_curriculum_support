# 第1章：TypeScriptとはなにか？

## この章で身につくこと

- TypeScript が「JavaScriptにルールを足した言語」だと分かる
- 「型」があると間違いに早く気づけることが分かる
- TypeScript が動くまでの流れ（変換→実行）が分かる

## 1-1 TypeScript は「ルール付き JavaScript」

TypeScript は、JavaScript に「ルール」を足した言語です。
そのルールの中心が「型」です。
「ここには文字だけ」「ここには数字だけ」と決められます。

JavaScript を知らなくても大丈夫です。
まずは「間違いが減る書き方」を学ぶところから始めます。

## 1-2 型があると何がうれしい？

型があると、間違いを早い段階で気づけます。

```ts
const price: number = "1000";
// エラー: 文字は number に入れられない
```

`price` という名前の箱は数字しか保存できないのに、
数字ではないデータを入れようとしているので、
書いた瞬間に間違いに気づけます。

## 1-3 TypeScript はそのままでは動かない

TypeScript のコードは、そのままではブラウザで動きません。
一度 JavaScript に変換してから実行します。

- TypeScript を書く
- JavaScript に変換する
- 変換された JavaScript が動く

「書く → 変換 → 動く」だけ覚えておけば OK です。

## 1-4 TypeScript はどんな場面で使う？

- チームで開発するとき（ミスが減る）
- 画面や機能が多いとき（迷いにくい）
- はじめての人でも安心して書きたいとき

小さな練習でも、
「間違いをすぐに見つけたい」ときに役立ちます。

## 1-5 今回のゴール

まずは「型があると安心して書ける」感覚がつかめれば OK です。
次の章で、よく使う基本の型からゆっくり覚えていきます。
