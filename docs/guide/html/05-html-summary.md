# 第5章：HTMLのまとめ

## この章で身につくこと
- HTMLでできること/できないことを整理できる
- タグの役割を思い出せる
- 「ここまででOK」と安心できる
- 次のCSS編にスムーズにつなげられる

## HTMLでやってきたことのおさらい
HTMLは「ページの中身・構造」を作るためのものです。
見た目を整えるものではありません。

これまで、タグで役割を決めることを学んできました。
見出し、文章、リンク、リストなど、意味のある部品を作ってきました。

## HTMLでできること / できないこと
できることは「中身を用意すること」です。

- 文章を書く
- リンクを作る
- 入力欄や選択肢を置く
- 情報を表やリストで整理する

できないことは「見た目を自由に整えること」です。

- 見た目を自由に整える
- 横並び・中央寄せなどのレイアウト制御

これらはCSSの役割です。
「できないことがあるのは普通」と覚えておけばOKです。

## HTMLだけで作る簡単なページ（総合デモ）
見出し、文章、リンク、リスト、選択肢、表まで入れた最小ページです。

```html
<!doctype html>
<html lang="ja">
<head>
  <meta charset="UTF-8" />
  <title>HTMLまとめデモ</title>
</head>
<body>
  <h1>HTMLのまとめ</h1>
  <p>HTMLはページの中身と構造を作ります。</p>
  <a href="#">リンクはこちら</a>
  <ul>
    <li>見出し</li>
    <li>文章</li>
    <li>リンク</li>
  </ul>
  <select>
    <option>選択肢A</option>
    <option>選択肢B</option>
  </select>
  <table>
    <tr>
      <th>項目</th>
      <th>内容</th>
    </tr>
    <tr>
      <td>役割</td>
      <td>中身を作る</td>
    </tr>
  </table>
</body>
</html>
```

## 完璧じゃなくていい、ここまでで十分
全部覚えていなくてOKです。
「意味」「役割」「どこに書くか」が分かっていれば十分です。
困ったら公式を見れば大丈夫です。

## 次はCSS編へ
HTMLは部品を作るものです。
次は、それをどう並べて、どう見せるかを学びます。
CSS編では「なぜ横に並ばないのか」「なぜ見た目が変わらないのか」を回収します。

## ❌ 初学者がよくやるミス
- 現象: HTMLだけで見た目を整えようとする → 原因: HTMLとCSSの役割を分けていない → 直し方: HTMLは中身、見た目はCSSと分ける
- 現象: タグを全部覚えようとしてしまう → 原因: 暗記が必要だと思い込む → 直し方: よく使うタグだけ覚え、困ったら公式を見る
- 現象: できない＝向いてないと思ってしまう → 原因: 最初はできないのが普通だと知らない → 直し方: 今の理解で十分、次の章でできることが増える

ここまで来た時点で、HTMLのスタートラインはしっかり越えています。
