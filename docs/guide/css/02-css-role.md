# CSSは何を担当する？
〜構造（HTML）と見た目（CSS）の分離〜

## 1. この章で身につくこと
- HTMLとCSSの役割の違いが分かる
- なぜ役割を分けるのかが分かる
- 実務でこの考え方が重要な理由が分かる

## 2. HTMLとCSSを混ぜると何が起きる？
HTMLで見た目を調整しようとすると、修正が大変になります。
例えば、同じ色やサイズを何度も書くことになり、
後から直したいときに全部探して直す必要が出ます。

「今は動く」けれど「後で直しづらい」状態になります。

## 3. 正しい役割分担
- HTML：文章・構造・意味
- CSS：色・大きさ・余白・配置

HTMLは「何があるか」を決めるものです。
CSSは「どう見せるか」を決めるものです。

```html
<div class="card">
  <h1>タイトル</h1>
  <p>内容の文章です。</p>
</div>
```

```css
.card { border: 1px solid #333; padding: 12px; }
```

## 4. 「レイアウト」もCSSの仕事
横並びや中央寄せ、余白の調整もCSSの役割です。
このあたりの詳しいやり方は、後の章で解説します。

## 5. なぜ分けると良いのか
分けると、修正がとても楽になります。
見た目だけをまとめて変更できるからです。

また、構造と見た目が分かれていると、
画面の見通しが良くなり、実務で必須の考え方になります。

## 6. まとめ
HTMLとCSSは役割が違います。
CSSは見た目を一手に引き受けます。
