# 第8章：コンソールってなに？

## この章で身につくこと

- console が「確認のための出力」だと分かる
- console.log の使いどころが分かる

## 8-1 console は確認のためのメモ

プログラムが今どうなっているかを確認する場所です。
画面に出さない内部の値を見られます。

## 8-2 console.log の使い方

```ts
const price = 1200;
console.log(price);
```

`price` の中身をコンソールに表示します。
「今いくらになってる？」を確認できます。

## 8-3 どんなときに使う？

- 変数に何が入っているか知りたい
- 条件分岐が動いているか確認したい
- エラーの原因を探したい

「画面に出す」ではなく「自分が確認するため」と覚えると分かりやすいです。

## ❌ よくあるミス

- コンソールを開いていないのに「出てこない」と思う
- `console.log` を残したまま本番コードにしてしまう
- コンソールのエラーを無視して原因が分からなくなる
