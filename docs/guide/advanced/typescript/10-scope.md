# 第10章：ブロックスコープとグローバルスコープ

## この章で身につくこと

- スコープが「使える範囲」だと分かる
- ブロックスコープとグローバルスコープの違いが分かる

## 10-1 スコープは「使える範囲」

変数には「使える範囲」があります。
その範囲をスコープと呼びます。

## 10-2 ブロックスコープ

`{}` の中で作った変数は、外では使えません。

```ts
if (true) {
  const message = "Hello";
  console.log(message);
}

// console.log(message); // 外では使えない
```

`message` は `if` の中だけで使えます。
外では見えません。

## 10-3 グローバルスコープ

ファイルの一番外で作った変数は、
そのファイル全体で使えます。

```ts
const appName = "My App";

const showName = (): void => {
  console.log(appName);
};
```

`appName` はファイル内ならどこからでも使えます。

## 10-4 なぜスコープが大事？

- 変数の衝突を防げる
- どこで使われるか分かりやすい

まずは「外に出られない箱がある」と覚えれば OK です。

## ❌ よくあるミス

- ブロックの外で変数を使ってしまいエラーになる
- `var` を使って意図しない場所でも使えてしまう
- 同じ名前の変数を増やして混乱する
