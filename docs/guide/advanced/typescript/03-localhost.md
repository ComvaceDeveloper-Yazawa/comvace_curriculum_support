# 第3章：ローカルホストってなに？

## この章で身につくこと

- ローカルホストが「自分のPCの中」だと分かる
- 開発中の画面がブラウザで見える理由が分かる

## 3-1 ローカルホストは自分のPC

ローカルホスト（localhost）は、
「自分のPCの中にあるサーバー」のことです。

インターネットではなく、
自分のPCの中で動いています。

## 3-2 なんでブラウザで見えるの？

開発用のサーバーを起動すると、
PCの中で「ページを配る役」が動きます。

その結果、ブラウザで
`http://localhost:5173` のように表示できます。

- `localhost`: 自分のPC
- `5173`: 番号（ポート）

この番号が「入り口の番号」だと思ってください。

## 3-3 本番公開との違い

- ローカルホスト: 自分だけが見える
- 公開済みURL: 誰でも見える

今は「自分のPCで動かして確認する場所」と覚えれば OK です。

## ❌ よくあるミス

- 開発サーバーを起動していないのに `localhost` を開いてしまう
- ポート番号が違っていて画面が出ない（例: 5173 ではなく別の番号）
- 「本番URLと同じもの」と勘違いしてしまう
