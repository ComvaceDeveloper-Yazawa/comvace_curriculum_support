# 第21章：ミスの読み方（TSエラー・実行時エラー）

## この章で身につくこと

- エラーは「場所と理由」を教えてくれると分かる
- TypeScriptエラーと実行時エラーの違いが分かる

## 21-1 TypeScriptのエラー

```ts
const price: number = "1000";
```

「文字を number に入れられない」と教えてくれます。
書いた瞬間に間違いを見つけられるのが強みです。

## 21-2 実行時エラー

```ts
const titleEl = document.querySelector("h1");
console.log(titleEl.textContent);
```

`titleEl` が `null` だとエラーになります。
実行したときに起きるのが実行時エラーです。

## 21-3 エラーの読み方

- どのファイル/行か
- 何が原因か
- どう直すか

この順番で読むと、落ち着いて直せます。

## 21-4 最初は「行番号だけ見る」でもOK

慣れるまでは「どこで起きたか」だけ見れば大丈夫です。
少しずつ内容を読めるようになればOKです。

## ❌ よくあるミス

- エラーの行番号を見ずに別の場所を直してしまう
- 1つ目のエラーだけで止まってしまい原因を見失う
- 実行時エラーと型エラーを同じものだと思ってしまう
