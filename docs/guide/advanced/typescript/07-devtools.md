# 第7章：検証ツールの使い方（DevTools）

## この章で身につくこと

- 検証ツールが「ブラウザの道具箱」だと分かる
- Elements / Console / Network の役割が分かる

## 7-1 検証ツールってなに？

ブラウザに付いている「開発用の道具箱」です。
見た目・エラー・通信をチェックできます。

ショートカット例:
- Mac: `⌥ + ⌘ + I`
- Windows: `F12` または `Ctrl + Shift + I`

## 7-2 よく使う3つ

### Elements

HTML と CSS を見て調整できます。
「今どの要素がどんな見た目か」を確認します。

### Console

エラーやログを見る場所です。
プログラムが動いているか確認できます。

### Network

通信の結果を見る場所です。
APIがちゃんと返っているかを確認します。

## 7-3 最低限覚える使い方

- 見た目がおかしい → Elements
- 動かない → Console
- データが来ない → Network

最初はこの使い分けだけで OK です。
