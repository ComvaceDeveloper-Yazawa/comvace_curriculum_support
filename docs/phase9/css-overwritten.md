# CSSが打ち消される理由（優先順位、上書き）

## 結論
CSSは「強いものが勝つ」「後に書いたものが勝つ」が基本です。

## 何が起きているか
同じ要素に複数のCSSが当たると、より強いルールが残ります。だから「書いたのに効かない」が起きます。

## 悪い例（デモ）
弱い指定だと、後から書いたルールに負けます。

デモ: `/demos/css-overwritten.html`

```css
p { color: red; }
.title { color: blue; }
```

## 良い例（デモ）
狙いを絞った指定にすると、意図どおりになります。

デモ: `/demos/css-overwritten.html`

```css
p.title { color: blue; }
```

## なぜそうなるか
CSSには「詳細な指定が強い」というルールがあります。タグより`class`、`class`より`id`が強いです。

## よくあるミス
- 同じCSSを別の場所で上書きしていることに気づかない
- `!important`で無理やり解決しようとする
