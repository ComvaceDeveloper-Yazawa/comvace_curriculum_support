# リセットCSS（なぜ必要か、最低限の方針）

## 結論
ブラウザの初期スタイルを整えると、ズレが減ります。最低限のリセットだけで十分です。

## 何が起きているか
ブラウザは最初から`margin`や`font-size`を持っています。これが「同じHTMLなのに違って見える」原因になります。

## 悪い例（デモ）
デフォルトの`margin`が残っていて、余白が勝手に入ります。

デモ: `/demos/reset-css.html`

```css
body { margin: 8px; }
```

## 良い例（デモ）
最低限、`margin`をゼロにします。

デモ: `/demos/reset-css.html`

```css
* { margin: 0; padding: 0; box-sizing: border-box; }
```

## なぜそうなるか
ブラウザごとに初期スタイルが違うため、リセットで差をならします。

## よくあるミス
- 強すぎるリセットでフォームが崩れる
- リセットを入れずに「なぜずれるの？」になる
