# レスポンシブ対応のやり方
〜PCとスマホ、両方で「きれいに見せる」ための考え方〜

## 1. この章で身につくこと
- レスポンシブ対応とは何かが分かる
- なぜ必要なのかが分かる
- PC基準で作る理由が分かる
- メディアクエリの基本的な意味が分かる
- 実務でよくやる調整内容が分かる

## 2. レスポンシブ対応って何？
レスポンシブ対応とは、
「画面サイズが変わっても、読みやすく・使いやすく表示すること」です。

PCとスマホでは横幅も情報量も操作性も違います。
同じ見た目のままだと、文字が小さくなったり、
要素が詰まって読みにくくなります。

## 3. なぜレスポンシブ対応が必要なの？
PCではきれいでも、スマホだと崩れることがよくあります。
横幅が狭くなると、要素が重なったり、文字が読みにくくなります。

「PC＝正解」ではありません。
スマホで見る人が多い現実があるので、
両方で読める状態を作る必要があります。

## 4. この教材でのレスポンシブ対応ルール
- PC：1440px
- スマホ：390px
- それ以外は考えない
- まずは「この2点で崩れない」ことが目標

## 5. レスポンシブ対応の基本的な考え方
結論：PCを基準に作ります。

PCを先に作る理由は、
情報量が多く、設計のベースになるからです。

スマホ用は「必要な部分だけ上書き」する考え方です。
全部作り直すのではなく、
変えたいところだけをスマホ条件で調整します。

## 6. メディアクエリってなに？
メディアクエリは、
「この条件のときだけCSSを当てる仕組み」です。

例えば、`@media (max-width: 390px)` は
「画面の横幅が390px以下のときに、
この中のCSSを使ってね、という意味です。」

ここはコードで見た方が早いです。

```css
.title {
  font-size: 28px;
}

@media (max-width: 390px) {
  .title {
    font-size: 20px;
  }
}
```

つまり「スマホサイズのときだけ、見た目を変える」ための仕組みです。

## 7. max-width ってなに？
`max-width: 390px` は
「画面が390px以下になったら」という条件です。

pxは画面の幅の単位です。
スマホサイズの目安として390pxを使っています。

## 8. min-width ってなに？
`min-width: 390px` は
「画面が390px以上のときに」という条件です。

PC側だけにCSSを当てたいときに使います。
「この幅より小さくなったら使わない」という線引きだと思ってください。

```css
.title {
  font-size: 20px;
}

@media (min-width: 390px) {
  .title {
    font-size: 28px;
  }
}
```

## 9. 実際によくやるレスポンシブ調整（コード例あり）

### ① wrapper の左右余白をスマホで小さくする
PCでは余白が広くてOKです。
スマホでは狭くする必要があります。
wrapper を使うと1箇所で調整できます。

```css
.wrapper {
  padding: 0 80px;
}

@media (max-width: 390px) {
  .wrapper {
    padding: 0 16px;
  }
}
```

### ② 商品一覧の列数を変える（grid）
PCは4列、スマホは2列にします。

```css
.product-list {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
}

@media (max-width: 390px) {
  .product-list {
    grid-template-columns: repeat(2, 1fr);
  }
}
```

### ③ 横並びを縦並びに変える（flex）
ツールバーなどはスマホで縦並びにします。

```css
.products-toolbar {
  display: flex;
  justify-content: space-between;
}

@media (max-width: 390px) {
  .products-toolbar {
    flex-direction: column;
  }
}
```

## 10. よくある間違い
- すべてをメディアクエリで書こうとする
- 最初からスマホ用を作る
- HTMLを書き換えてしまう

## 11. 面接での説明例
「PCを基準に作って、
390px以下でメディアクエリを使って
余白やレイアウトを調整しています」

## 12. まとめ
レスポンシブ対応は特別な技術ではありません。
設計の延長です。

wrapper / grid / flex を組み合わせて考えると、
実務でも整理しやすくなります。
